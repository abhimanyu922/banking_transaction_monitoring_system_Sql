# # Banking Transaction Monitoring System  


## Project Overview
This project simulates a **realistic banking transaction monitoring system** that detects suspicious behavior, fraud patterns, account takeover signals, and financial anomalies.



## Dataset / Tables Included

| Table Name            | Description |
|----------------------|-------------|
| customers            | Customer profile, identity, and demographic data |
| branches             | Bank branches with location metadata |
| accounts             | Customer account details, balance, type, and branch |
| cards                | Debit/credit card info tied to accounts |
| merchants            | Merchant registry (POS / ecommerce) |
| transactions         | Central fact table with 5000+ banking transactions |
| logins               | Login/IP/device trails for fraud correlation |
| fraud_alerts         | Alerts generated by rule-based or model-based systems |
| transaction_labels   | Ground-truth fraud labels for ML training |
| transfers            | Inter-account transfers |


# 20 Business KPIs 
1. What is the total number of transactions per day?
2. How many debit vs credit transactions occurred?
3. What is the total debit value (bank revenue outflow)?
4. What is the average transaction amount per customer?
5. Who are the top 10 highest-spending customers?
6. What is the total revenue per merchant?
7. How many total transactions does each customer perform?
8. What is the transaction volume across different channels (UPI/ATM/POS)?
9. What is the city-wise merchant revenue distribution?
10. How many transactions succeeded/failed/pending?
11. What is the average account balance per account type?
12. Which merchants processed the highest number of transactions?
13. What is the monthly transaction volume?
14. What is the daily average transaction amount?
15. What is the total incoming credit per account?
16. What is the total outgoing debit per account?
17. What is the total balance per branch?
18. Which customers have multiple cards?
19. What are the top 10 highest-value transactions?
20. How many login attempts does each customer make?

# 25 Fraud Detection SQL Rules 
1. Did any account perform more than 10 transactions in 5 minutes?
2. Are there any unusually large transactions above 50,000?
3. Are there transactions occurring between 12 AM–4 AM?
4. Are customers using too many IP addresses?
5. Are customers using too many devices?
6. Is the same IP used by multiple customers?
7. Are there high-risk merchant categories involved?
8. Which accounts have too many failed transactions?
9. Did any account suddenly spend more than 100,000 in a day?
10. Are there repeated small transactions (“structuring”)?
11. Is the card used across multiple countries on the same day?
12. Is the same card used in two far locations in < 15 minutes?
13. Any customers with high login failure attempts?
14. Same card mapped to multiple accounts?
15. Merchants accessed by large number of customers (risk hotspot)?
16. Unusual refund patterns detected?
17. Any customer accessing multiple accounts?
18. Accounts with too many fraud alerts?
19. Cards used in 5+ cities (possible stolen card)?
20. Accounts with excessive transfers?
21. Accounts with excessive ATM withdrawals?
22. Accounts with unusually high average transaction amounts?
23. High-risk cities for transactions?
24. Customers with extremely frequent logins?
25. Accounts with too many refunds?
